# 🚀 Metodologia Ágil no Projeto

Este projeto utiliza práticas de **Scrum** e **Kanban** para organizar tarefas, melhorar a colaboração e garantir entregas contínuas.

---

## 📅 Scrum - Estrutura

- **Papéis**
  - Product Owner: responsável pela visão do produto
  - Scrum Master: facilita o processo e remove impedimentos
  - Time de Desenvolvimento: executa as tarefas

- **Eventos**
  - Sprint Planning: definição das metas da sprint
  - Daily Stand-up: reunião rápida para alinhamento
  - Sprint Review: apresentação dos resultados
  - Sprint Retrospective: análise do que pode melhorar

- **Artefatos**
  - Product Backlog: lista de funcionalidades desejadas
  - Sprint Backlog: tarefas da sprint atual
  - Incremento: resultado entregue ao final da sprint

---

## 📊 Kanban - Fluxo de Trabalho

O quadro Kanban é usado para visualizar o progresso das tarefas:

| Status        | Descrição                                    |
|---------------|----------------------------------------------|
| 📥 To Do      | Tarefas planejadas, ainda não iniciadas      |
| 🔄 In Progress| Tarefas em andamento                         |
| ✅ Done       | Tarefas concluídas                           |

---

## 🎯 Benefícios

- Transparência no andamento do projeto
- Melhor comunicação entre os membros
- Entregas contínuas e incrementais
- Foco em melhoria contínua
